# 🔧 주문 기능 빠른 테스트 가이드

## 🚀 즉시 확인 방법

### 1. 브라우저 접속

- URL: http://localhost:5174
- 화면 상단에 "장바구니: 0개" 표시 확인

### 2. 메뉴 추가 테스트

1. **카테고리 선택**: "커피" 탭 클릭
2. **메뉴 선택**: 아메리카노 카드 클릭
3. **모달 확인**: 메뉴 상세 정보가 팝업으로 표시
4. **담기**: "4,500원 담기" 버튼 클릭
5. **결과 확인**:
   - 헤더에 "장바구니: 1개"로 변경
   - 우측 장바구니에 아메리카노 표시
   - 모달 자동 닫힘

### 3. 주문하기 테스트

1. **주문 버튼**: 우측 장바구니 하단의 "주문하기" 버튼 클릭
2. **로딩 상태**: 버튼이 "주문 중..."으로 변경 (1초간)
3. **성공 알림**: 화면 상단에 녹색 알림 "주문이 성공적으로 접수되었습니다! 🎉"
4. **장바구니 초기화**: 헤더가 "장바구니: 0개"로 변경, 우측 장바구니 비움

## 🐛 문제 발생 시 체크

### 문제 1: 헤더에 장바구니 개수가 안 보임

→ 페이지 새로고침 (F5)

### 문제 2: 메뉴 클릭 시 모달이 안 열림

→ 브라우저 Console (F12) 에서 오류 확인

### 문제 3: "담기" 버튼 클릭 후 장바구니에 안 들어감

→ Console에서 "장바구니에 메뉴 추가: ..." 메시지 확인

### 문제 4: "주문하기" 버튼이 안 보임

→ 장바구니에 메뉴 먼저 추가 필요

### 문제 5: "주문하기" 클릭 시 반응 없음

→ Console에서 "주문하기 버튼 클릭됨" 메시지 확인

## 🎯 성공 시나리오 전체 플로우

```
1. 페이지 로드 ✅
   ↓
2. 카테고리 선택 (커피) ✅
   ↓
3. 메뉴 선택 (아메리카노) ✅
   ↓
4. 모달에서 담기 ✅
   ↓
5. 장바구니 확인 (1개) ✅
   ↓
6. 주문하기 버튼 클릭 ✅
   ↓
7. 로딩 (1초) ✅
   ↓
8. 성공 알림 표시 ✅
   ↓
9. 장바구니 초기화 (0개) ✅
```

## 📱 실제 테스트 시연

**지금 바로 테스트하려면:**

1. 브라우저에서 http://localhost:5174 열기
2. F12로 개발자 도구 열기 (Console 탭 확인)
3. 위 단계 따라하기
4. 각 단계마다 Console 메시지 확인

**예상 Console 로그:**

```
장바구니에 메뉴 추가: 아메리카노 수량: 1
장바구니 추가 완료
주문하기 버튼 클릭됨
장바구니 아이템: [{menu: {...}, quantity: 1}]
주문 생성 시도: [{menuId: 1, quantity: 1}]
주문 성공: {id: 123, totalPrice: 4500, ...}
```

---

> **💡 팁**: 헤더의 "장바구니: X개" 숫자를 보면서 테스트하면  
> 메뉴 추가/제거가 정상 작동하는지 쉽게 확인 가능합니다!
